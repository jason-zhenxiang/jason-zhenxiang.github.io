#### 半导体RAM存储器
- 静态RAM存储器
- 动态RAM存储器（Dynamic RAM）
  - 256M比特单元的组织
    - 将$32M \times 8$转为$16K \times 16K$
    - $\bar{RAS}和\bar{CAS}$表示低电平响应
![32mX8的组成.png](/resources/计算机组成/32mX8的组成.png)
  - 快速页模式（Fast Page Model）
    - 读、写电路还作为**锁存器**，选择同一行地址时不用重新选定行，行地址缓存在锁存器中，直接给列地址就可以读取数据 
    - 速度更快，可以传输更多数据
  - 同步DRAM（Synchronous DRAM）
    - 操作信号与一个**时钟信号**同步
    - 首先是一个DRAM，不读不写也会漏电，所以需要**Refresh counter**
    - 列地址锁存器变成了**列地址计数器**，能够实现自增（随着时钟信号递增）
![SDRAM.png](/resources/计算机组成/SDRAM.png)
    - Burst Operation
      - SDRAM具有多种操作模式，能够通过向模式寄存器中写入一些信息来选择操作模式
      - 块操作能够读取大块数据
      - 模式寄存器中有一个12-bit的值，SDRAM会通过查看这个值确定每次Burst操作会连续读取多少列
      - 例：最上方clock为时钟，RAS低电平时将地址总线上的行地址加载到行地址锁存器中进行译码（实际中会需要多个时钟周期），选择一行，CAS低电平时将地址总线中的地址加载到列地址技术器当中进行译码、翻译，选择一列；然后从第一列开始连续地读取四列，此时列地址计数器自增（相比快速页模式，不用再给列地址）
![块读取例子.png](/resources/计算机组成/块读取例子.png)
    - 延迟和带宽Latency and Bandwidth
      - 延迟和带宽是用来评估存储系统表现的参数
      - 对于大块数据传输，延时用来表示开始传输数据第一个字之前所需要的的时间
      - 上一张图中延迟是**5个时钟周期**
      - 带宽是指访问存储器的时候，每秒钟可以传输（读或写）的位数或字节数
      - 存储部件的带宽与**访问数据的速度**有关，也与**并行访问的位数**有关
      - 有效带宽 = 总线速度 * 总线带宽
    - Double-Data-Rate SDRAM
      - 在时钟周期的上升沿和下降沿都传输数据，加倍了数据传输率，一个时钟周期可以传两个数据
      - 选择行、列的延迟没有变化，但是带宽翻倍 


#### 大型储存器的结构
- **静态内存系统（Static Memory System）**
  - 使用的芯片规格一致
  - **位扩展法**：芯片每个存储单元的字数足够，但**位数小于存储器字长**，需要进行**位扩展**
  - 例：用$1M\times 1$位存储芯片组成$1M\times 8$位的存储器
    - 共需要8片$1M\times 1$位存储芯片，每片存储同一位权的**一位数据**（八个位来自不同的芯片，一个芯片是一位）
    - 访问芯片需要20位地址码
    - 需要读写信号R/W
    - 并排画，画一行八列的图像
  - 如何计算需要几块及其排布？字数除字数得m，位数除位数得n，得到排布为**m行n列**
  - **字扩展法**：芯片每个存储单元得位数等于存储器字长，但是**容量（字数）不够**，需要进行**字扩展**
    - 例：用$256K\times 8$位存储芯片组成$1M\times 8$位的存储器
    - 同样字数除字数，位数除位数确定排布
    - 芯片内部选择一个字需要18位地址，整个存储器需要20位地址，多的两位用来进行chip select（选择具体从哪一个芯片读取），使用2：4译码器选择
    - 画图技巧：数据总线画在下面
  - **字位同时扩展法**：单个芯片的字数和位数均小于主存的容量要求，需要进行字、位扩展
    - 例：用$512K\times 8$位存储芯片组成$2M\times 32$位的存储器
    - 同样字数除字数，位数除位数确定排布、数量要求
    - 片选信号每次选择一行
    - 注意数据总线的编号，地址总线的连接（需要连到每一个字上）
    - **考试画阵列有分**
  - 总的来说，大容量存储器最重要的就是确定芯片的排布及需要的芯片的数量，

- 动态内存系统 Dynamic Memory System
  - 动态内存系统的组成与静态内存系统的方法几乎一样，除了封装方式不同，动态内存系统以存储器模块为单位进行组织
  - 由DRAM构造，在计算机中充当主存
  - 分为SIMM、DIMM两种封装方式
    - SIMM：早期系统中DRAM的封装方式，可以单面有芯片或者双面都有芯片
    - DIMM：当下主流的封装方式，能够一次性传递64位信息

#### 存储层次结构 Contents of this lecture
- 理想的存储器：访问时间极快、无限大容量、价格便宜
  - 问题：三个要求之间相互矛盾，越大的存储器读取越慢、越快的存储器价格越贵
  - 用同类型储存器构造理想存储器是不可能的，因为总是有满足不了的要求
- 计算机中的存储分层结构：
  - **这里放一张结构图**
- 存储结构的基本思想
  - Buffer：小而快的存储设备，一般作为下层大容量速度慢的存储设备的暂存区域（上层做下层的缓冲）
    - 全部数据存储在最低级别的内存中
    - 数据在相邻层之间通过拷贝传输
  - General Buffering Concepts：程序需要目标d，d存在一些块b中
    - Hit：程序在k层的缓冲中找到了b
    - Miss：b不在第k层中，所以k需要从下层中复制到k层
- 存储结构的工作原理
  - 使用局部性原理（the Principle if Locality）
    - 程序在运行时不会随机访问整个内存地址空间，而是在某一时刻，只集中访问很小的一部分地址
    - 时间局部性 Temporal Locality：最近访问的元素不久后还会访问
    - 空间局部性 Spatial locality：一个被访问的元素附近的元素不久后会被访问
  - 局部性原理就是将可能被访问的数据放到离CPU更近的地方
  
#### **Cache Memories**
- Cache完全是由**硬件（Hardware）**管理的
- **引入Cache的目的：**
  - **问题：**处理器的速度比DRAM快得多
  - Cache是一个小而快的SRAM-based内存，充当**主存和CPU**之间的缓冲
  - 储存被访问频率最高的数据
  - 主存和Cache之间数据传输的单位是**块（Block）**，Cache与CPU之间以字为单位读写
  - 目的：让主存的速度尽可能地接近最快的内存；同时，以较低地价格提供大的内存大小
- **Cache/Main-Memory系统的组织**
  - 按理来说，Cache line和block是不一样大的，Cache line还包括tag等其他信息，但是一般来说我们认为它们一样大
  - 块是事先划分好的，是字附近的一组数据
![主存的组织.png](/resources/计算机组成/主存的组织.png)
- **核心基础概念**
  - **Hit**：CPU要访问的信息在Cache中能找到（被缓存）
  - **Miss**：CPu要访问的信息不在Cache中，需要访问主存
  - **Hit rate**：要访问的信息在Cache中能找到的比例
  - **失效率**：1 - Hit rate
- **Cache读取操作**
  - CPU不需要显式地知道Cache的存在（对CPU是透明的=不要去管它）
  - CPU通过读写指令使用地址判断位置是否在主存中
  - **Read Request**
    - Hit情况：Cache中已经有这个字，把这个字发给CPU
    - Miss情况：两种方法
      - 先将包含这个字的块复制到Cache中，再把这个字发给CPU
      - 把这个块复制到Cache中的同时，把这个字转发给CPU
#### **Cache设计相关**
- **映射方案**
  - 决定块在一开始被复制到Cache中时会被**放置在哪里**；要求将主存的地址转换为Cache中的地址
  - 三种方法：**直接映射，全相连，组相连**
  - **参数定义：**
    - 主存块编号：$B_j, j = 0, 1, \ldots , n-1, n = 2^s$
    - Cache行编号：$L_i, i = 0, 1, \ldots , m-1, m = 2^r$
    - 每个块/行包含的字数：$k = 2^w$
    - 主存地址总长度：$s+w$位
  - **直接映射 Direct Mapping**
    - 每个主存中的块只能放在**一个**Cache的缓存行中
    - **映射函数**：$i = j mod m$
    - **地址划分**：主存地址被拆分为**3段**，从高位到低位依次是：Tag(s-r位)+Line(r位)+Word(w位)
      - Word位：定位块内的具体字
      - Line位：定位到唯一的Cache行
      - Tag位：和定位到的Cache行的Tag对比，匹配+有效位为1就命中，否则缺失
    - **优点**：简单、易实现；价格低
    - **缺点**：块的位置固定
  - **全相连 Fully Associative Mapping**
    - 无映射函数，主存中的块可以储存在Cache中的**任意行**
    - 主存地址被分为**两段**：Tag(s位)+Word(w位)，每个块的Tag不同
      - 没有Line位，因为块可以放在任意位置
      - 寻址时，需要把主存地址的Tag，和**Cache中所有行的Tag并行比较**，有匹配的就命中，否则缺失
    - **优点**：在选择cache的位置上有充分的自由
    - **缺点**：需要复杂的电路来并行检查所有缓存块的标签
  - **组相连 Set Associative Mapping**
    - Cache被划分为一系列的组，每组包含k个行，称为**k路组相联**
    - 一个块只能映射到**一个特定的组**，组内行任意放
    - **映射函数：**
      - Cache有v组，一共有k行
      - m = v * k；i = j modulo v
      - 特殊情况：v = m，k = 1，直接映射；v = 1，k = m，全相联
    - **地址划分**：主存地址被拆分为3段，Tag(s-d位)+Set(d位，$v=2^d$)+Word(w位)
    - **优点**：相比直接映射具有一定的自由度；相比全相联硬件更简单
    - **缺点**：需要k个比较器
- **块标识**
  - **有效位 Valid bit**：
    - 当**首次通电**时，Cache不包含有效信息（Cache内信息无意义），需要加上标记
    - 每个增行加一个位（例如0代表无效，1代表有效，从主存钟读出），用来标记数据是否有效
    - 初始时，每一行的有效位都是0
- **替换算法 Replacement Algorithms**
  - 几种选择方法：
    - **最近最少使用 LRU**：选择**最长时间没有被访问**的，把它换出去
      - 基于程序的局部性原理，最近被访问过的块，未来大概率还会被访问
      - 特点：命中率最高，是现代Cache的主流方案
    - **先进先出 FIFO**：替换掉Cache中**停留时间最长**的块
      - 特点：实现简单，但是可能把频繁访问的块替换掉，命中率低于LRU
    - **随机替换 Random**：随机选择一个块替换
- **写策略 Writing Policies**——可能考选择/简答，简答更多
  - **写命中 Write Hit**
    - **写直达 Write Through**：CPU写数据时，**同时更新Cache和主存**，二者始终保持一致
      - **优点**：实现简单，没有数据一致性的问题，无需额外硬件开销
      - **缺点**：每次写操作都需要**访问慢速的主存**，总线流量大，写性能极差，哪怕写命中也要付出主存访问的代价
    - **写回 Write Back**：CPU写数据时，**只更新Cache中的对应行**，同时给这个行设置一个**脏位（Dirty bit）**，标记这个行被修改过；当这个行要被替换出去时，把脏块写回主存
      - **优点**：多次写同一个快，只需要一次写回主存，大幅减少贮存访问次数；写性能极高，总线流量小，是现代CPU的主流方案
      - **缺点**：Cache和主存会出现数据不一致，需要额外的一致性机制处理I/O访存等场景；每个Cache需要额外增加一位脏位，有额外硬件开销
  - **写缺失 Write Miss**：CPU要写的字不在Cache中
    - **写不分配 No-Write Allocate**：直接把数据写入主存，不把对应块加载到Cache中
      - 一般与Write Through搭配
    - **写分配 Write Allocate**：先把对应块从主存加载到Cache中，再更新Cache里的数据
      - 一般与Write back搭配

#### 多层Cache架构
- 现代CPU通常采用3级Cache架构，层级越靠近CPU，速度越快、容量越小；层级越远，速度越慢、容量越大。
- 指令与数据Cache
  - 很多现代系统都将Cache分为独立的指令Cache和数据Cache
  - 优点：比集成Cache的访问时间更短；提高了并行性和命中率

### **虚存 Virtual Memory**
#### 虚存的核心定义与本质
- 在执行程序时，可能会遇到： 
  - 主存空间不足以放下一个程序
  - 需要同时允许多个程序，但是主存空间无法容纳全部
- **Overlay 覆盖段技术**
  - 程序员将程序划分为**多个片段**，需要使用哪个片段就把哪个片段加载到主存中
  - 覆盖段组的划分需要程序员手动划分
- **虚存的本质**
  - 是一个抽象概念，不是一个物理设备
  - 利用**硬盘**来扩充物理主存的大小，超过其原本的大小
  - **目标：**让每个过程都认为有一个很大的内存
- **虚存相关术语**
  - **物理内存 Physical Memory**：计算机中真实存在的DRAM，真实的硬件存储空间
  - **虚拟/逻辑内存**：操作系统**让程序以为自己拥有**的内存空间
  - **物理地址**：物理存储器里真实的地址，对应真实存储
  - **虚拟/逻辑地址**：程序编写、运行时使用的地址，必须经过翻译才能变成物理地址
- **MMU 内存管理单元**
  - 在硬件层面完成虚拟地址到物理地址的翻译，翻译的依据是根据操作系统管理的**页表**
  - 虚拟地址对CPU是透明的，翻译工作完全由MMU硬件自动完成
- 虚拟内存的优势
  - 提供**巨大的地址空间**，解决了主存不够用的问题
  - **简化重定位**，同一个程序可以在物理存储的任意位置开始
  - **加快程序启动速度**，不用把程序的所有代码和数据加载到主存后再启动，只需要加载当前要用的部分，剩余的用到了再加载
  - 解放程序员，不需要程序员手动写覆盖逻辑
#### 虚存的核心实现：Paging 分页
- 分页的基础定义——信息交流的基本单位
  - **页 Page**：虚拟地址空间中，一段固定大小、连续的地址块，是主存和磁盘之间数据传输的最小单位（虚拟页）
  - **页框 Page Frame**：主存中，一段和页大小完全一致的连续存储空间，一个页框刚好能放下一个虚拟页（物理页）
  - 放置策略：只使用**全相联**，与Cache的全相联规则完全一致
- **分页的地址结构**
  - 虚拟地址和物理地址，都被拆分为高位的**页号字段**和低位的**页内偏移字段**
  - **虚拟地址**：虚拟页号（VPN）+页内偏移（Offset）
    - **VPN**：决定虚拟地址属于哪一个虚拟页
    - **Offset**：决定这个地址再页内的具体位置
  - **物理地址**：物理页号（PPN）+页内偏移（Offset）
- **页表 Page Table**
  - 页表是**页表项**的数组，记录虚页和实页之间的对应关系
  - 页表是以虚页号为**索引**，虚页号不在页表的内容当中
  - 页表储存在主存当中，每个进程都有自己独立的页表，在主存的不同位置
  - **页表基址寄存器**：记录当前进程页表在主存里的起始地址，MMU通过这个寄存器找到页表
- 页表项的结构
  - **物理页号**：如果页在主存里，就存对应的PPN；不在主存里，就存这个页在磁盘上的存储位置
  - **有效位**：标记这个虚拟页是否已经加载到物理内存中（1=在主存，0=不在主存）
  - **修改位**：标记这个页被加载到主存后，是否被修改过（写策略为write-back）
  - **使用位/引用位**：标记这个页有没有被访问过，用于LRU页替换算法
  - **访问控制位**：标记这个页的权限，可读、可写、可执行等，用于内存保护
- **页命中 Page Hit**
  - CPU要访问的虚拟地址对应的页，已经加载到物理主存里了
- **页故障 Page Fault**
  - CPU要访问的虚拟地址对应的页，不在物理主存里，有效位为0
- **分页的不足与优化**
  - 不足：内存是存在主存里的，CPU每做一次访存，都要访问两次主存
    - 第一次：访问主存里的页表，完成地址翻译，拿到物理地址
    - 第二次：用物理地址访问主存，拿到需要的数据
  - **旁路缓冲器 TLB（快表）**：是集成在MMU里的一个小型高速Cache，是实际存在的硬件，专门缓存**最近常用的页表项PTE**
    - TLB是基于内容的比较
    - TLB可以看作页表的Cache
  - TLB的工作流程：
    - CPU发出虚拟地址，拆分为VPN和Offset
    - MMU先把VPN送到TLB里查找，而不是直接去主存查页表
    - TLB Hit：
    - TLB Miss：
- **页替换算法**
  - LRU、FIFO、操作系统实现的其他方法
- **写策略**：只使用写回（Write Back）算法，因为硬盘CPU无法直接访问
- **分页方式的缺点：内部碎片化**
  - 页是固定大小的，而程序的大小几乎不可能正好是页大小的整数倍，所以程序的最后一个页，大概率不会被填满，剩下的空闲空间就被浪费了，这就是内部碎片
  - **量化规律**：如果页大小是n字节，一个程序平均会浪费n/2字节的内部碎片
- **页大小的权衡**
  - **页表大小**：页表大小与页大小成反比，页越大，页表项越少，页表占用的主存也少；页越小，页表项越多，页表占用的主存越多
  - **磁盘传输效率**：大页的批量磁盘传输效率更高，一页能传输更多数据；小页的磁盘传输效率低，同样的数据量需要多次IO
  - **内部碎片：页越大**，内埔平均碎片浪费越多；页越小，内部平均碎片浪费越少，内存利用率越高
### **辅助存储 Secondary Storage**
#### 机械硬盘的机械结构
- **存储介质**
  - **核心组成**：一个或多个圆形盘片（Platter），安装在同一个主轴上，主轴带动所有盘片以恒定速度高速旋转
  - **盘片材质**：金属或塑料基底，表面涂有可磁化的磁性薄膜，二进制数据就是通过磁膜的不同磁化方向来存储的
  - **读写特性**：盘片通常是**双面可读写**的，一个盘片对应两个盘面，每个盘面都可以存储数据
- **读写头 Read/Write Head**
  - **工作原理**：每个读写头由磁轭和磁化线圈组成
    - **写数据**：给不同线圈不同极性的电流脉冲，改变盘面磁膜的磁化方向，记录0和1
    - **读数据**：磁膜的磁场变化会在线圈里感应出电流，还原出存储的二进制数据
  - **固定头方案**：每个磁道对应一个独立的读写头，磁头固定在刚性支架上，完全不用移动
    - **优点**：无寻道开销，访问速度极快
    - **缺点**：成本极高，容量做不大，磁头数量太多
  - **可移动头**：每个盘面仅配一个读写头，所有盘面的磁头都安装在同一个可移动的磁臂上
    - **优点**：结构简单、成本低，盘片可以做很多面，容量上限高
    - **缺点**：需要移动磁头定位，产生寻道时间开销

- 温切斯特硬盘（Winchester Hard Disk）
  - 由IBM公司在1968年研发
  - **特性**：盘片、读写头、驱动机构全部蜜蜂在空气过滤的洁净腔体内
  - **核心优势**：
    - 1.密封环境无灰尘污染，读写头可以离盘面更近，磁道可以做得更密，数据存储密度大幅提升；
    - 2.相同体积下，密封设计可以做到更大的容量，同时可靠性大幅提升
#### 硬盘的数据组织
- **磁道与扇区（Track and Sector）**
  - **磁道**：每个盘面上，划分出的一圈圈同心圆，就是磁道，数据就存储在这些同心圆的磁化区域里
    - **特性**：磁道从外向内编号，最外侧的是0磁道（硬盘的引导区、分区表都存在0磁道，是硬盘的核心区域）；现代硬盘单盘面有500~2000个磁道，磁道宽度仅$0.1~0.2\mu m$
  - **扇区**：每个磁道被等分成一个个扇形的区域，就是扇区。扇区是硬盘读写的最小物理单位，计算机不能单独读写半个扇区，最少也要读写一个完整的扇区
    - 传统硬盘扇区固定为512字节，现代高级硬盘采用4KB扇区，和虚拟内存的页大小对齐
    - **数量**：传统硬盘每个磁道的扇区数固定（10~100个），大容量硬盘采用等密度记录，外圈磁道的扇区数更多，最大化利用盘面空间
  - **扇区的完整结构**：扇区不只有用户能使用的512字节的数据，还包括一些其他信息
    - 1.**前导码**（Preamble/扇区头）：存储扇区的地址标识（柱面号、盘面号、扇区号），硬盘控制器通过它识别目标扇区
    - 2.**数据区**：用户可用的512字节数据
    - 3.**EGC纠错码**：存储校验数据，用来检测和纠正读写过程中出现的位错误，保证数据可靠性
    - 4.**扇区间隙**（Inter-sector Gap）：相邻两个扇区之间的空白区域，用来区分扇区，避免读写时的信号干扰
![扇区结构图.png](/resources/计算机组成/扇区结构图.png)
- **柱面（Cylinder）**
  - **定义**：所有盘面、同一半径的所有磁道，共同组成 一个逻辑柱面。比如0号柱面就是所有盘面的0号磁道组成的圆柱
  - 可移动磁头的所有磁头是同步移动的——磁臂移动到某个位置时，所有盘面的磁头，都整好对准自己盘面的、同一半径的磁道，也就是同一个柱面
  - **优势**：访问同一个柱面内的所有数据，完全不需要移动磁臂，没有寻道时间开销
- **硬盘的物理地址结构**：硬盘的物理地址包含三个维度：盘面号、磁道号/柱面号，扇区号，这是硬盘定位数据的唯一物理地址
  - **盘面号**：选择哪个盘面的磁头工作，对应**选磁头**，电子切换，无机械开销
  - **磁道号/柱面号**：控制磁臂移动到目标柱面，对应**找磁道**。机械移动，产生寻道时间，是最大的开销
  - **扇区号**：等待盘片旋转，让目标扇区转到磁头下方，对应**找扇区**，机械旋转，产生旋转延迟
- **磁盘格式化**
  - **核心动作**：把空白盘片划分成磁道和扇区，写入每个扇区的前导码、ECC码，同时扫描并标记出有缺陷的坏扇区、坏磁道，避免系统往里面写数据
  - **容量损耗**：格式化写入的管理信息，会占用硬盘总容量的约$15\%$，这就是我们买的硬盘标称容量和系统里显示的可用容量有差距的核心原因之一
#### 数据访问与性能衡量
- **磁盘访问的完整流程**：
  - 等待设备空闲->等待总线可用->寻道->旋转延迟->数据传输
- 磁盘访问时间
  - **访问时间 = 寻道时间 + 旋转延迟时间**
  - **寻道时间**
    - **定义**：把读写磁头移动到目标磁道/柱面所需要的时间，是硬盘访问最大、最不稳定的开销
    - **核心特性**：寻道时间和移动的磁道数成正比，相邻磁道的寻道时间极短，从最内圈到最外圈的全行程寻道时间最长
    - 现代机械硬盘的平均寻道时间为$5~8ms$
  - **旋转延迟时间**
    - **定义**：磁头已经定位到目标磁道后，等待目标扇区的起始位置旋转到磁头下方所需要的时间
    - **核心特性**：和硬盘的转速成反比，转速越快，旋转延迟越短；常用平均旋转延迟来衡量，也就是盘片旋转半圈的时间
- **数据传输的性能指标**
  - **传输率（Transfer Rate）**：磁头定位到目标扇区后，数据连续读写的速度，单位通常是MB/s；他和它和硬盘的转速、磁道的位密度成正比
  - **传输时间（Transfer TIme）**：传输目标数据快本身所需要的时间，等于**数据块大小/传输率**，通常只有零点几毫秒，远小于寻道时间和旋转延迟
