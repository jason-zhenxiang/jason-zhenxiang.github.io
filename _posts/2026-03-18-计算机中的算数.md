## **一、整数表示**
- 计算机中的整数可分为**有符号数**与**无符号数**，其中有符号数有**原码、反码、补码**三种表示形式
  - **原码**：第一位是**符号位**，其余位为**数值位**
    - n-bit的原码可表示范围为$$-(2^{n-1}-1) \le V(B)\le (2^{n-1}-1)$$
  - **反码**：正数的反码就是其原码，负数的反码是由其数值所对应正数**按位取反**获得的
    - 例：$+18 = 0010010$, $-18 = 11101101$
    - n-bit的反码可表示范围为$$-(2^{n-1}-1) \le V(B)\le (2^{n-1}-1)$$
    - 问题：$+0$和$-0$的表示不一样
  - **补码**：正数的补码就是其原码，负数的补码是由其数值所对应正数**按位取反后加一**获得的
    - 例：$+18 = 0010010$, $-18 = 11101110$
    - n-bit的反码可表示范围为$$-2^{n-1} \le V(B)\le (2^{n-1}-1)$$
    - 优点：0的表示方式唯一，在计算机中最常使用
- **不同长度比特的数值表示转换：**
  - 原码：将符号位移动至**最左边**的位置，然后中间位置**填充0**
    - 例：$+18 = 00010010$(8 bits), $+18 = 0000000000010010$(16 bits)
  - 补码：将符号位移动至**最左边**的位置，然后中间位置使用符号位进行填充
    - 例：$-18 = 11101110$(8 bits), $-18 = 1111111111101110$(16 bits)

## **二、有符号数的加减**
- **一位二进制整数的加减法**
![一位正二进制加法.png](/resources/计算机组成/一位正二进制加法.png)
- **n-bit有符号数的加法**（补码形式）
  - 将符号位视为**最大有效位（MSB）**，逐位相加，并忽略最大有效位的进位
  - 只要计算结果在$[-2^{n-1}, 2^{n-1}-1]$之间，和的结果在数值上就是正确的
  - **溢出（Overflow）**指的是算术操作的结果不在可表示范围内，所导致的错误
    - *e.g.*&ensp; $(0101)_2 + (0100)_2 = (1001)_2$，结果为$-7$，但是显然正确的结果为$+9$
    - **充要条件：**当且仅当两个**同符号**的数相加，得到的结果的符号与这两个数的符号**相反**时，会产生溢出
    - *note：* 符号位的进位不能作为溢出的判断条件
- **n-bit有符号数的减法**（补码形式）
  - 要对两个数字X和Y进行减法运算（即计算$X-Y$），先求出Y的**补码**，再将其与X相加，具体操作遵循加法规则
    - 求补码操作：按位取反，末位加一
  - 只要计算结果在$[-2^{n-1}, 2^{n-1}-1]$之间，差的结果在数值上就是正确的
  - 因为减法可以转换为加法，故也会有溢出的情况
  - **减法规则的本质：**$X-Y=X+(-Y)$

## **三、行波加法器与加减法逻辑单元**
- **一位全加器**（1-bit full adder）：一个全加器需要**三个比特**的输入，然后输出一个**两位比特**的输出（包含**本位和、进位**）
![一位全加器.png](/resources/计算机组成/一位全加器.png)
  - **逻辑表达式**：
    - \begin{aligned}
S_i &= \bar{x}_i\bar{y}_ic_i + \bar{x}_iy_i\bar{c}_i + x_i\bar{y}_i\bar{c}_i + x_iy_ic_i 
&= x_i \oplus y_i \oplus c_i
\end{aligned}
    - \begin{aligned}
c_{i+1} = y_i c_i + x_i c_i +x_i y_i
\end{aligned}

- **n位行波加法器**(Ripple-Carry Adder)：从LSB开始计算，像波传递一样，逐位计算
  - 以**四位加法器**为例：
    - 组成：内部由四个**全加器**组成，依次串联
    - 输入：两个4-bit的数字，一个初始进位
    - 输出：一个4-bit的和，一个进位 
![四位行波加法器.png](/resources/计算机组成/四位行波加法器.png)
  - **n位行波加法器**内部由n个全加器依次串联组成，需要两个n-bit数字、一个初始进位作为输入；输出一个n-bit的和，一个进位
  - 当且仅当两个加数同号、和的符号与加数相反时会发生**溢出**

- **门延迟**（Gate Delays）：
  - 每个逻辑门电路在收到输入之后和输出结果之间都有一小段时间，这段时间称为**门延迟**
  - 在计算机组成这门课程中，认为所有逻辑门的**门延迟时间相同**

- **行波加法器中的延迟传播**
  - 在一个组合逻辑电路中，总延迟由网络中**最长的逻辑门路径的延迟决定**
  - 行波加法器得到如下结果的延迟时间：
    - $c_{n-1}: 2(n-1)T$，
    - $s_{n-1}: 2(n-1)T+T=(2n-1)T$，
    - $c_{n-1}: 2(n-1)T+2T=(2n)T$
- **加减法逻辑单元硬件组成**
  - 减法可以转换为加法，用同一个加法器既可以实现加法也可以实现减法，为了实现这个功能我们需要对$Y$做处理，使其**使用一套电路实现加减法**
  - 已知：$0\oplus Y=Y, 1\oplus Y=\bar{Y}$，故减法情况下先**使用一个异或门**得到$\bar{Y}$，再加一就可以转化成加法
  - 加减控制：加法时，初始进位$c_0 = 0$，$Y$与0异或，仍然是$Y$；减法时，初始进位$c_0 = 1$，$Y$与1异或，结果得到$\bar{Y}$
  - 门延迟：多了一个异或门对$Y$进行处理：
    - $c_{n-1}: (2n-1)T$，
    - $s_{n-1}: 2(n-1)T+T=(2n)T$，
    - $c_{n}: 2(n-1)T+2T=(2n+1)T$
![加减法逻辑单元.png](/resources/计算机组成/加减法逻辑单元.png)

## **四、快速加法器的设计** （Fast Adders）
  - 行波加法器的门延迟过长，可能的解决方法：
     1. 使用更快的电子技术来实现更短的门延迟——硬件技术发展较慢
     2. 使用**增强的逻辑门电路**结构（更加可行）
  - **进位的产生函数与传播函数**
    - $s_i = x_i \oplus y_i \oplus c_i$
    - $c_{i+1}=y_i c_i+x_i c_i+x_i y_i=x_i y_i + (x_i+y_i)c_i$
    - 对逻辑表达式进行替换：
      - $G_i = x_i y_i$，进位的**产生函数**，只要$G_i=1$就会有进位
      - $P_i = x_i + y_i$，进位的**传播函数**
      - $c_{i+1}=G_i+P_i c_i$
    - 对替换后的表达式做进一步简化：$P_i = x_i + y_i \rightarrow P_i = x_i \oplus y_i$
      - 原因：简化前后只有$x_i=1, y_i=1$时真值不同，但此时$G_i=1$，$c_{i+1}=1$，$P_i$对整体真值无影响，所以可以进行替换。
      - 目的：为了与$s_i$共用一个异或门 
  - Bit-stage cell结构：
![B_cell.png](/resources/计算机组成/B_cell.png)
    - 门延迟：一个B-cell的延迟是$2T$，所有的$x_i, y_i$都同时到达；经过一个T的时间得到所有的$G_i, P_i$，再经过一个T得到$s_i$
  - **进位产生**：所有的进位$c_i$都可以用$G_i, P_i, c_0$表示
    - \begin{aligned}
c_{i+1} &= G_i + P_i c_i 
= G_i + P_i (G_{i-1} + P_{i-1} c_{i-1}) 
= G_i + P_i G_{i-1} + P_i P_{i-1} c_{i-1} 
\quad \cdots 
= G_i + P_i G_{i-1} + P_i P_{i-1} G_{i-2} + \ldots + P_i P_{i-1} \ldots P_1 G_0 + P_i P_{i-1} \ldots P_0 c_0
\end{aligned}
    - 得到$G_i, P_i$后，在经过$2T$即可得到所以的$c_i$
  - **总延迟**：
    - 得到所有进位位：$3T$
    - 得到最终的和：$4T$
  - *e.g.：***四位Carry-Lookahead Adder**
![四位CLA.png](/resources/计算机组成/四位CLA.png)

  - **扇入(Fan-in)限制**
    - **扇入**：一个逻辑门输入的个数
    - **扇出**：一个逻辑门输出作为另一个逻辑门输入的个数
    - 现实中扇入扇出系数**不能过大**（最大不能超过8、9，5、6是比较合适的），这会限制逻辑门的速度
    - 因此，构建32位加法器需要8个4位加法器进行级联，门延迟也会叠加，最终的延迟为：$(3+2*7)T=17T$（获得$s_{31}$），$(3+2*7+1)T=18T$（获得$c_{32}$）
      - 第一个$3T$中包含了获得所有$G_i, P_i$的一个$T$，后续每个B_cell需要两个$T$

## **五、无符号数的乘法**
#### **1.手动乘法（Manual Multiplication Algorithm）**
- 例：$M = 1101, Q = 1011$，计算$P = M \times Q$
![手动乘法.png](/resources/计算机组成/手动乘法.png)
- Multiplucand：被乘数，Multiplier：乘数，Partial Products：部分积，Product：乘积
- 两个n-bit的整数相乘，乘积的位数最多为2n位

#### **2.矩阵乘法（Array Multiplication）**
- 通过数字逻辑电路。实现类似手动乘法的操作
![矩阵乘法.png](/resources/计算机组成/矩阵乘法.png)
- **缺点：**逻辑门开销过大，两个n-bit的数字相乘需要$n(n-1)$个全加器，$n^2$个与门

#### **3.顺序乘法（Sequential Multiplication）**
- 通过一个**顺序电路**和一个**n-bit加法器**，也可以实现两个n-bit数字的乘法
- 初始时，$C$，$A$都为0，$M$中保存被乘数，$Q$中保存乘数，计数器的值设置为$n$；若$Q_0=1$，则$A = A + M$，然后右移$C$，$A$，$Q$；若$Q_0=0$，就直接进行右移操作，然后计数器减一；重复该步骤直至计数器的值为零
![顺序乘法流程.png](/resources/计算机组成/顺序乘法流程.png)

## **六、有符号数的乘法**
- 对于有符号数，上一部分中的几种乘法无法处理含一个或两个负数的乘法，需要使用其他方法
- **布斯算法**可以直接处理有符号数的乘法，它对乘数进行**重新编码**，该方法也适用于补码
- 布斯算法的优势在于，对于含有连续的0、1的二进制串，它能够使**部分积的数量减少**
- 对乘数的重新编码：
![布斯算法重编码.png](/resources/计算机组成/布斯算法重编码.png)
- 记背方法：找二进制串中**多个（包括一个）连续的1**和**这些1前面的0**
  - 前面的0 $\rightarrow$ +1；中间的1 $\rightarrow$ 0；后面的1 $\rightarrow$ -1
- **硬件实现：**
  - 被乘数和乘数分别存放在寄存器M和寄存器Q中
  - 在寄存器Q的最低有效位（$Q_0$）的逻辑右侧额外设置了一个一位寄存器，记为$Q_{-1}$
  - 控制逻辑逐位扫描乘数的每一位；在检查每一位时，会同时检查其右侧相邻的位
  - 乘法运算的最终结果储存在**寄存器A**和**寄存器Q**中
![布斯算法流程图.png](/resources/计算机组成/布斯算法流程图.png)
- **控制逻辑**：通过**相邻两位的组合判**断操作，而非逐位乘加；每次迭代都会检查$Q_0$（Q寄存器当前最低位）和$Q_{-1}$（$Q_0$右侧的辅助位）
  - $Q_0 Q_{-1} = 01: A = A + M$（加被乘数）
  - $Q_0 Q_{-1} = 10: A = A - M$（减被乘数）
  - $Q_0 Q_{-1} = 00/11$：无操作
![布斯算法计算.png](/resources/计算机组成/布斯算法计算.png)
- **布斯算法的优点：**
  - 能够解决正数和负数的乘法
  - 当乘数有较多连续的1时，计算效率高
